SERVICE DESK TICKETING SYSTEM - SETUP INSTRUCTIONS
==================================================

STEP 1: Create the Analysts Table
----------------------------------
Run the SQL script to create the analysts table:

File: create_analysts_table.sql

This will:
- Create the analysts table with columns: id, username, password_hash, full_name, email, is_active, created_datetime, last_login_datetime
- Create an index on username for faster lookups
- Insert a default admin user (username: admin, password: admin123)

IMPORTANT: Change the admin password after first login!


STEP 2: Access the Login Page
------------------------------
Navigate to: login.php

Default credentials:
- Username: admin
- Password: admin123


STEP 3: View the Inbox
-----------------------
After logging in, you'll be redirected to: inbox.php

The inbox interface includes:
- Header with welcome message and logout button
- Left panel: Email list with refresh button
- Right panel: Reading pane for viewing selected emails


FILES CREATED
-------------
1. create_analysts_table.sql - SQL script to create the analysts table
2. login.php - Login form with authentication
3. inbox.php - Main Outlook-style interface
4. analyst_logout.php - Logout handler for analysts
5. api/get_emails.php - API endpoint to fetch email list
6. api/get_email_detail.php - API endpoint to fetch full email details


FEATURES
--------
- Secure login with password hashing (bcrypt)
- Session management
- Outlook-style interface with message list and reading pane
- Real-time email selection and display
- Auto-marking emails as read when opened
- Responsive design
- Refresh button to reload emails


NEXT STEPS
----------
1. Run the SQL script in SQL Server Management Studio or your preferred tool
2. Test the login at login.php
3. Change the default admin password
4. Add more analysts as needed:
   INSERT INTO analysts (username, password_hash, full_name, email)
   VALUES ('username', PASSWORD_HASH, 'Full Name', 'email@example.com');

   To generate password hash in PHP:
   echo password_hash('your_password', PASSWORD_BCRYPT);


SECURITY NOTES
--------------
- All passwords are hashed using bcrypt
- Sessions are used for authentication
- SQL injection protection via prepared statements
- XSS protection via htmlspecialchars in output
- Remember to change default credentials!
